# 🐍 Python Chapter 8: 函数 (Functions) 速查表

> **核心概念**：如果说之前的程序是“顺流而下”的流水线，那这一章就是教你如何打造“专属工具箱”。把重复的代码封装成一个个带有名字的工具（函数），随时调用，隐藏复杂的细节，让代码更简洁、更模块化。

---

## 1. 定义函数与传递参数 (Defining & Passing Arguments)

**生活类比**：去快餐店点餐。函数就像服务员，你把需求（实参）告诉他，他去后厨执行，最后把结果端给你。

| 功能 | 英文关键字/语法 | 代码示例 | 备注/易错点 |
| :--- | :--- | :--- | :--- |
| **定义与形参** | `def` | `def greet(name):` | **形参**：定义时的“占位符”（如 `name`）。 |
| **位置实参** | (按顺序对应) | `describe_pet('hamster', 'harry')` | ⚠️ **陷阱**：顺序极其重要！如果反了，就会变成“名字叫仓鼠的宠物 harry”。 |
| **关键字实参** | `key=value` | `describe_pet(name='harry', animal='dog')` | **必做动作**：明确写出变量名，此时**顺序无关紧要**，代码更安全。 |
| **默认值** | `param=value` | `def make_shirt(size='L'):` | **默认配置**：调用时不传尺码，默认发大号 (L)。 |

---

## 2. 返回值与传递列表 (Return Values & Lists) [🔴重难点]

**生活类比**：`print()` 只是把菜端出来给你**看一眼**，而 `return` 是把做好的菜**打包交到你手上**，你可以带回家继续加工（用于后续代码逻辑）。

### 核心区别：修改列表的原件与复印件

| 操作 | 代码思路 | 典型应用 / 生活类比 |
| :--- | :--- | :--- |
| **返回字典** | `return {'first': f, 'last': l}` | **组装数据**：把零散的姓名、年龄打包成一个完整的字典返回。 |
| **修改原列表** | `def print_models(unprinted):` <br> `unprinted.pop()` | **交出钥匙**：函数直接操作原列表，**原件会被改变（甚至清空）**。适合流水线搬运作业。 |
| **传递副本 (切片)** | `print_models(unprinted[:])` | **给复印件**：在列表后加 `[:]`，函数只在副本上修改，**原始数据绝对安全**。 |

💡 **记忆口诀**：
* **不想被改，就加切片** `[:]`。

---

## 3. 任意数量的参数 (Arbitrary Arguments)

**生活类比**：去吃自助餐，服务员不知道顾客会拿几个盘子，必须准备一个“大托盘”来无限制地接收所有盘子。

| 方式 | 语法/魔法符号 | 收集后的数据类型 | 典型场景 |
| :--- | :--- | :--- | :--- |
| **任意位置实参** | `*args` | **元组 (Tuple)** | `def make_pizza(*toppings):` <br> 顾客可能加1种配料，也可能加5种。 |
| **任意关键字实参** | `**kwargs` | **字典 (Dictionary)** | `def build_profile(**user_info):` <br> 收集用户任意的附加信息（身高、爱好等）。 |

⚠️ **参数排序铁律**：如果一个函数里有多种参数，顺序必须是：**普通参数 -> `*args` -> `**kwargs`**。

## 4. 模块导入 (Importing Modules)

**核心概念**：把写好的函数放在单独的 `.py` 文件中，主程序通过 `import` 把它们“借”过来用，保持主界面清爽。

| 导入方式 | 语法示例 | 调用方式 | 备注 |
| :--- | :--- | :--- | :--- |
| **导入整个模块** | `import pizza` | `pizza.make_pizza()` | 最安全，一看就知道函数来自哪里。 |
| **导入特定函数** | `from pizza import make_pizza` | `make_pizza()` | 最简练，不用写前面的模块名。 |
| **给函数起别名** | `from pizza import make_pizza as mp`| `mp()` | 解决原函数名太长，或者与本地变量重名的问题。 |
| **导入所有函数** | `from pizza import *` | `make_pizza()` | ❌ **极度危险**：容易覆盖当前文件的同名代码，不推荐！ |

## 5. ⚠️ 常见坑点与防坑指南 (Troubleshooting - PEP 8)

* **默认值形参的顺序报错 (SyntaxError)**：
  * 现象：定义函数时直接标红报错 `non-default argument follows default argument`。
  * 原因：带有默认值的形参（如 `age=18`）放在了普通形参的前面。
  * 急救：**把所有带默认值的形参，统统挪到括号的最右边！**

* **等号两边的空格 (PEP 8 规范)**：
  * 现象：代码能跑，但显得“不专业”，或者有代码格式化警告。
  * 规范：在指定**默认值**或使用**关键字实参**时，等号 `=` 两边**绝对不要加空格**。
  * 正确：`def function(param='default'):` 或 `fn(name='kris')`
  * 错误：`def function(param = 'default'):`

* **丢失原始列表数据**:
  * 如果你发现调用完某个函数后，你的原列表变成了空列表 `[]`。
  * 急救：检查调用时是否忘记加切片 `[:]`，赶紧改成 `function_name(my_list[:])`。
